    Bots v0.1.0 Bots.Telegram.Processor

    Декодирует json-сообщения, делегирует готовые объекты нужному commander и отправляет ответ на сервер telegram.

    Модуль Processor декодирует приходящее в виде json-сообщение от сервера telegram в структуру Bots.Telegram.TelegramResponse в соответствии с официальной документацией Bot API. После парсинга структура передаётся в отдельный поток, где передаётся в commander, заданный в опциях бота. После получения ответа от commander происходит отправка подготовленных данных на сервер telegram.

    Functions

    decode_response_data(data, options)

    Декодирование json-данных, пришедших в ответ на запрос методом long-polling. Данные декодируются в структуру Map. После успешного декодирования результат отправляется в отдельный поток со списком функций process_messages. Более того, этот метод возвращает максимальный идентификатор объекта update среди пришедших. Этот метод принимает два входящих параметра: data и options.

    Параметр data - это json-данные, полученные посредством getUpdates от сервера telegram.

    Второй параметр options содержит commander и token бота.

    В случае получения ошибочных данных от сервера telegram, метод возвращает соответствующее сообщение ”bad response”. Если в ответе от сервера telegram пришли валидные данные, но они не имеют новых сообщений, то decode_response_data выдает сообщение ”no new message”.

    decode_webhook_data(data, options)

    Метод предназначен для декодирование json-данных, пришедших методом webhook. Данные декодируются в структуру Map. После успешного декодирования результат отправляется в отдельный поток с функцией process_messages.

    В качестве входящего параметра принимает переменную data содержащую json-данные. Второй параметр options содержит commander и token бота.

    prepare_and_send(commander, update, token)

    Метод prepare_and_send является методом подготовки сообщения для отправки. Выполняет получение ответа от commander и передачу его в метод отправки. Метод принимает 3 входящих параметра: commander, update и token.

    Параметр commander указывает на то, с каким commanderом необходимо работать.

    Параметр update содержит Map-объект сообщения.

    Параметр token содержит соответственно маркер (идентификатор) бота.

    process_messages(data, options)

    Метод для параллельной обработки сообщений. После обработки сообщение передает данные в метод подготовки для отправки сообщений. В качестве входящих параметром принимает два переменных: data и options.

    В options должен быть представлен commander для обработки сообщения и token для отправки ответа. Каждое сообщение из массива выполняется в своём потоке.

    Параметр data содержит Map-объект декодированный из json-данных, пришедших с сервера telegram.
